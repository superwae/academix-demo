نظرة عامة على المنصة

AcademiX منصة تعليمية متكاملة (LMS) مصممة لإدارة التعلم الإلكتروني بثلاث بوابات رئيسية: الطالب، المدرّس، والمسؤول. المنصة مبنية على معمارية حديثة وواجهة مستخدم قابلة للتخصيص بالكامل.

  • للطالب: لوحة تحكم، كتالوج الدورات، الفصول، الدروس، التقويم، الواجبات، الاختبارات، الرسائل، الملف الشخصي
  • للمدرّس: لوحة تحكم، إدارة الدورات والدروس، الواجبات والاختبارات، قائمة الطلاب المعرضين للخطر، الرسائل
  • للمسؤول: لوحة تحكم شاملة، إدارة المستخدمين والدورات، التقارير والرسوم البيانية، سجلات التدقيق، المالية (المعاملات، المدفوعات، توزيع الإيرادات)

الذكاء الاصطناعي والتحليلات محرك رئيسي

١. نظام التوصيات الذكي (Recommendation Engine)

نظام توصيات يعتمد على خوارزميات التصفية التعاونية (Collaborative Filtering) والتصفية المعتمدة على المحتوى (Content-Based Filtering) — جميع العمليات داخلية بدون اعتماد على APIs خارجية.

  • توصيات شخصية: بناءً على تاريخ التسجيل، التصنيفات، والأهداف التعليمية
  • دورات مشابهة: لدورة معينة بناءً على التصنيف والوسوم
  • الدورات الرائجة: حسب معدل التسجيل والشعبية
  • واصل التعلم: استكمال الدورات قيد التقدم
  • خرائط التصنيفات: ربط اهتمامات المستخدم من الـ onboarding مع تصنيفات الدورات (مثل Technology → Programming, Cloud, DevOps)

٢. تحليلات الأداء والتنبؤ (Analytics & Predictions)

خدمة تحليلات متقدمة تعتمد على القواعد الإحصائية والتنبؤية — دون الحاجة لـ ML APIs خارجية:

  • درجة المخاطر (Risk Score): تحديد الطلاب المعرضين للخطر بناءً على: عدم النشاط (7 أيام تحذير، 14 يوم خطر)، التقدم المنخفض (<25%)، الدرجات المنخفضة (<60%)، انسحاب الدورات
  • التنبؤ بالدرجة النهائية (Predicted Final Grade): حساب تنبئي يعتمد على متوسط الدرجات، درجة المشاركة، ونسبة الإكمال
  • درجة المشاركة (Engagement Score): قياس تفاعل الطالب مع المحتوى والواجبات والاختبارات
  • توصيات نصية تلقائية: نصائح وإجراءات مقترحة لكل طالب حسب وضعه (مثل: "راجع الدروس الأخيرة"، "حسّن حضورك")
  • تقارير إحصائية للدورات: معدل الإكمال، معدل الرسوب، متوسط التقدم، معدل تسليم الواجبات

٣. التقارير والرسوم البيانية (Admin Reports)

  • رسوم بيانية للتسجيلات، النشاط، اتجاهات الإكمال، أداء الدورات
  • قسم الطلاب المعرضين للخطر مع تصدير CSV
  • لوحة مالية مع KPIs ومقاييس الإيرادات

المحادثة والملاحظات والإشعارات

١. نظام المحادثة (Messaging)

تقنية SignalR — اتصال فوري (Real-time)

  • محادثات مستخدم لمستخدم (User-to-User) — ليست بوت ذكاء اصطناعي
  • Hub: /hubs/messaging
  • نقل LongPolling في التطوير لتجنب مشاكل WebSocket مع الـ proxy
  • إعادة اتصال تلقائي: [0, 2s, 5s, 10s]
  • Timeout السيرفر: 30 ثانية

٢. الملاحظات (Notes)

ملاحظات مزامنة مع توقيت الفيديو

  • أثناء الدرس: إضافة ملاحظات مرتبطة بوقت معين في الفيديو (timestamp)
  • النقر على الملاحظة: ينقل الفيديو تلقائياً إلى نفس التوقيت
  • التخزين: localStorage (academix_lesson_notes)
  • البحث: بحث في محتوى الملاحظات حسب الدورة
  • ترتيب: حسب التوقيت داخل الدرس، أو حسب تاريخ التحديث ضمن الدورة

٣. الإشعارات (Notifications)

أنواع الإشعارات

  • assignment — واجب
  • exam — اختبار
  • announcement — إعلان
  • grade — درجة
  • message — رسالة
  • deadline — موعد نهائي

الميزات: مزامنة مع API عند المصادقة، Fallback على localStorage للعمل غير المتصل، حد أقصى 100 إشعار محفوظ، واجهة NotificationBell

المعمارية التقنية

نموذج التصميم

Monolith Modular — API واحد قابل للنشر، مع حدود نظيفة بين الطبقات. ليس Microservices.

الهيكل المعماري للـ Backend

معمارية Clean Architecture بأربع طبقات واضحة:

١. Domain (النطاق)

الدور: قلب المنطق — الكيانات والثوابت فقط، بدون اعتماد على أي طبقة خارجية.

  • BaseEntity: أساس مشترك (Id, CreatedAt, UpdatedAt, IsDeleted, DeletedAt) — تدقيق وتليين الحذف
  • Enums: CourseLevel, Modality, CourseStatus, EnrollmentStatus, AssignmentStatus, QuestionType, ConversationType
  • الكيانات: User, Role, Course, Enrollment, Lesson, Assignment, Exam, Conversation, Message, Notification, Tag, Review, UserInterest, RefreshToken

٢. Application (التطبيق)

الدور: حالات الاستخدام، المنطق التجاري، والتعريفات (Interfaces).

  • Interfaces: IApplicationDbContext, IAuthService, ICourseService, IRecommendationService, IAnalyticsService, IEmailService, IStorageService...
  • Services: AuthService, CourseService, EnrollmentService, RecommendationService, AnalyticsService, NotificationService, ExamService, ConversationService...
  • DTOs: نقل البيانات بين الطبقات (LoginRequest, CourseDto, AnalyticsDto...)
  • Common: Result<T>, PagedResult<T> — نمط النتائج بدون استثناءات

٣. API (واجهة البرمجة)

الدور: استقبال الطلبات، التوجيه، المصادقة، والتشغيل (Hosting).

  • Controllers: Auth, Users, Courses, Enrollments, Assignments, Exams, Lessons, Analytics, Recommendations, Notifications, Export, Health
  • Middleware: GlobalExceptionHandler، RequestLogging، SecurityHeaders
  • SignalR Hub: MessagingHub — المحادثات الفورية
  • التكوين: JWT، CORS، Swagger، Rate Limiting، Health Checks

٤. Infrastructure (البنية التحتية)

الدور: تنفيذ التفاصيل التقنية — قاعدة البيانات والخدمات الخارجية.

  • ApplicationDbContext: EF Core + PostgreSQL — تنفيذ IApplicationDbContext
  • Migrations & Configurations: إعدادات الجداول والعلاقات (UserConfiguration, CourseConfiguration...)
  • الخدمات: JwtService، MailjetEmailService، LocalStorageService
  • DatabaseSeeder: بيانات تجريبية عند التشغيل

تبعيات الطبقات: Domain مستقلة ← Application تعتمد على Domain ← Infrastructure تنفذ واجهات Application ← API تستضيف وتجمع كل الطبقات

الواجهة الأمامية (Frontend)

  • React 19 + TypeScript
  • Vite 7 — أداة بناء سريعة
  • React Router DOM 7
  • Zustand — إدارة الحالة
  • SPA — تطبيق صفحة واحدة

الواجهة الخلفية (Backend)

  • ASP.NET Core 8 Web API
  • معمارية الطبقات: API → Application → Domain ← Infrastructure
  • JWT + Refresh Tokens
  • SignalR للمحادثات الفورية

بنية الـ API

REST مُنسّق: api/v{version}/[controller] — النسخة الافتراضية v1

  • api/v1/courses — الدورات
  • api/v1/analytics/... — التحليلات
  • api/v1/recommendations — التوصيات
  • api/v1/notifications — الإشعارات
  • /hubs/messaging — Hub المحادثات
