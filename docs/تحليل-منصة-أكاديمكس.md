# تحليل منصة أكاديمكس (AcademiX) — دليل شامل

## ١. نبذة تعريفية

**أكاديمكس (AcademiX)** منصة تعليمية إلكترونية (LMS - Learning Management System) تدعم إدارة التعلم من طرف واحد: الطلاب والمدرسين والإداريين. توفر المنصة:

- **بوابات منفصلة** حسب الدور: طالب، مدرّس، مسؤول.
- **إدارة كاملة** للمقررات، الدروس، الواجبات، الامتحانات، والتسجيل في الشعب (أونلاين، حضور، هجين).
- **التواصل** عبر الرسائل والمحادثات.
- **لوحة تحكم إدارية** تشمل المستخدمين، المقررات، التقارير، المالية، وسجلات التدقيق.

المنصة مبنية على **واجهة أمامية (React + Vite)** و**واجهة خلفية (ASP.NET Core)** مع قاعدة بيانات، وواجهات API موحّدة.

## ٢. ما الذي يمكنك فعله على المنصة؟ (حسب الدور)

### 🔹 كـ **طالب (Student)**

| الصفحة / المسار | ماذا تفعل |
|------------------|------------|
| **لوحة التحكم** `/student/dashboard` | عرض ملخصك: إحصائيات، المقررات، الواجبات القادمة، الامتحانات، جدول اليوم، التسجيل السريع، وأقسام "موصى لك" و"الرائج". |
| **كتالوج المقررات** `/student/catalog` | تصفح المقررات، البحث والتصفية (حسب التصنيف، النوع، اليوم)، وعرض تفاصيل كل مقرر. |
| **التسجيل في مقرر** | من تفاصيل المقرر أو الكتالوج: اختيار **شعبة** (أونلاين / حضور / هجين) مع أوقات الحصص والمكان، والتحقق من تعارض الجدول قبل التأكيد. |
| **صفوفي** `/student/my-classes` | قائمة المقررات المسجلة فيها، والدخول إلى دروس كل مقرر. |
| **دروس المقرر** `/student/my-classes/:courseId/lessons` | قائمة الدروس مع التقدّم (مكتمل / غير مكتمل)، وفتح كل درس. |
| **عرض الدرس** `/student/my-classes/:courseId/lessons/:lessonId` | مشاهدة محتوى الدرس (فيديو، نص)، وتحديث التقدّم. |
| **التقويم** `/student/calendar` | عرض جدول أسبوعي لمقرراتك وشعبك (أوقات الحصص مأخوذة من الشعب)، والنقر على حدث لتفاصيله. |
| **الواجبات** `/student/assignments` | قائمة الواجبات (حسب المقرر)، تقديم الحلول، وعرض الحالة والدرجات. |
| **الامتحانات والاختبارات** `/student/exams` | قائمة الامتحانات، الدخول للامتحان، حل الأسئلة، وإرسال المحاولة وعرض النتائج عند النشر. |
| **الرسائل** `/student/messages` | المحادثات مع المدرسين أو الطلاب، وإرسال واستقبال الرسائل. |
| **الملف الشخصي** `/student/profile` | صورة الغلاف، الصورة الشخصية، النبذة (Bio)، الشهادات، التخصصات/المجالات، والشارات (Badges). |
| **الإعدادات** `/student/settings` | **الحساب والأمان**: الاسم، البريد (للقراءة فقط)، كلمة المرور، حفظ التغييرات، حذف الحساب. إعدادات الإشعارات وثيم الواجهة وإعادة تعيين بيانات التجربة. |

| **الواجبات** `/teacher/assignments` | قائمة الواجبات لجميع المقررات، وإنشاء واجب جديد. |

| **طلاب مقرر** `/teacher/courses/:id/students` | قائمة الطلاب المسجلين في مقرر معين. |
| **الرسائل** `/teacher/messages` | نفس واجهة الرسائل للتواصل مع الطلاب. |
| **الملف الشخصي** `/teacher/profile` | نفس الملف الشخصي (غلاف، صورة، نبذة، شهادات، تخصصات، شارات). |
| **الإعدادات** `/teacher/settings` | نفس إعدادات الحساب والإشعارات والثيم. |

## ٣. المسارات العامة (بدون تسجيل دخول)

| المسار | الوظيفة |
|--------|---------|
| `/` | الصفحة الرئيسية (عرض المقررات، الدعوة للتسجيل أو الدخول). |
| `/login` | تسجيل الدخول. |
| `/register` | إنشاء حساب جديد. |
| `/courses` | كتالوج المقررات للزوار. |
| `/courses/:id` | تفاصيل مقرر للزوار. |

بعد تسجيل الدخول يتم توجيهك حسب دورك: طالب → `/student/dashboard`، مدرّس → `/teacher/dashboard`، مسؤول → `/admin/dashboard`.

## ٤. هيكل المنصة التقني

### الواجهة الأمامية (Frontend)

- **الإطار:** React 19 + TypeScript.
- **الأدوات:** Vite 7، Tailwind CSS، Radix UI، Framer Motion.
- **التوجيه:** React Router مع حماية المسارات حسب الدور (`RoleGuard`).
- **الحالة:** Zustand (مخزن التطبيق والثيم)، واتصال بالـ API للبيانات الحية.
- **الاتصال الفوري:** SignalR للمحادثات والإشعارات.

### الواجهة الخلفية (Backend)

- **الإطار:** ASP.NET Core 8 (API).
- **قاعدة البيانات:** Entity Framework Core مع migrations.
- **واجهات API أساسية:** نسخة API (مثلاً `api/v1/`) تشمل:
  - **Auth** — تسجيل الدخول والتسجيل.
  - **Users** — بيانات المستخدم وتحديث الملف.
  - **Courses** — المقررات، الشعب، أوقات الحصص.
  - **Enrollments** — التسجيل في المقررات/الشعب.
  - **Lessons** — الدروس والمحتوى.
  - **Progress** — تقدم الطالب في الدروس.
  - **Assignments** — الواجبات وتقديم الحلول.
  - **Reviews** — تقييمات الواجبات.
  - **Exams** — الامتحانات والمحاولات.
  - **Conversations** — المحادثات والرسائل.
  - **Recommendations** — مقررات موصى بها.
  - **Analytics** — تحليلات.
  - **Onboarding** — تفضيلات المستخدم الجديد.

### الشعب (Sections) والتقويم

- كل مقرر يمكن أن يحتوي على **شعب** (أونلاين، حضور، هجين).
- لكل شعبة: اسم، مكان/منصة، **أوقات حصص** (يوم، بداية، نهاية)، وعدد المقاعد.
- أوقات الشعب تظهر في **تقويم الطالب** وتُستخدم للتحقق من التعارض عند التسجيل.

## ٥. كيف تستفيد من هذا الملف؟

1. **مرجع سريع:** تعرف أي صفحة تفتح لأي عمل (تسجيل، واجبات، امتحانات، إعدادات، إلخ).
2. **التدريب:** شرح قدرات كل دور (طالب، مدرّس، مسؤول) للمستخدمين الجدد.
3. **التطوير:** معرفة المسارات والمكونات المرتبطة عند إضافة ميزة أو تصحيح خطأ.
4. **التوثيق:** استخدامه كأساس لوثائق المستخدم أو دليل الإدارة.
5. **التخطيط:** تحديد الثغرات (مثلاً: تقارير إضافية، إشعارات، تصدير) وربطها بالصفحات الحالية.

## ٦. ملخص سريع — أين أفعل ماذا؟

| أريد أن… | أين أذهب (مثال) |
|----------|------------------|
| أتصفح المقررات وأسجّل | طالب → كتالوج المقررات أو الصفحة الرئيسية. |
| أختار شعبة (أونلاين/حضور) وأتحقق من الجدول | عند النقر على "التسجيل" في تفاصيل المقرر → اختيار الشعبة. |
| أشاهد دروسي وأحدّث التقدّم | طالب → صفوفي → المقرر → الدروس. |
| أرى جدول حصصي | طالب → التقويم. |
| أقدّم واجباً أو أرى درجتي | طالب → الواجبات. |
| أقدّم امتحاناً أو أرى النتائج | طالب → الامتحانات. |
| أضيف مقرراً أو شعبة أو أوقات حصص | مدرّس → إنشاء مقرر / مقرراتي → تعديل → الشعب. |
| أضع واجباً أو امتحاناً | مدرّس → الواجبات / الامتحانات → إنشاء. |
| أعدّل اسمي أو كلمة المرور أو أحذف الحساب | أي دور → الإعدادات (الحساب والأمان). |
| أعدّل صورتي والنبذة والشارات | أي دور → الملف الشخصي. |
| أتحكم بالمستخدمين أو المالية أو التقارير | مسؤول → المستخدمون / المالية / التقارير. |
